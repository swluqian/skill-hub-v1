# Feature Specification: Skill Hub

**Feature Branch**: `001-skill-hub`  
**Created**: 2026-04-22  
**Status**: Draft  
**Input**: User description: "创建一个 Skill Hub 平台，参考 https://skillhub.cloud.tencent.com/，支持浏览发现 Skills、详情与安装指引、提交与审核管理、用户注册登录与个人中心"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 浏览和发现 Skills (Priority: P1)

用户访问 Skill Hub 首页，可以看到精选推荐、下载热榜、最近上新三个维度的 Skill 列表。用户可以通过搜索框按关键词搜索 Skill，也可以按分类标签筛选。点击"探索全部"可以进入完整的 Skill 列表页，支持分页浏览。

**Why this priority**: 浏览和发现是平台的核心价值，没有这个功能用户无法找到和使用任何 Skill，是 MVP 的最基础功能。

**Independent Test**: 无需登录即可访问首页和列表页，搜索并浏览 Skill 卡片，验证分类筛选和分页功能正常工作。

**Acceptance Scenarios**:

1. **Given** 用户首次访问首页, **When** 页面加载完成, **Then** 展示下载热榜、为你推荐、最近上新三个区块，每个区块展示若干 Skill 卡片（名称、简介、下载量、标签）
2. **Given** 用户在搜索框输入关键词, **When** 点击搜索或按回车, **Then** 展示匹配的 Skill 列表，支持按相关度排序
3. **Given** 用户在列表页选择分类标签, **When** 点击某个分类, **Then** 列表仅展示该分类下的 Skill
4. **Given** Skill 列表超过一页, **When** 用户滚动到底部或点击分页, **Then** 加载下一页数据

---

### User Story 2 - Skill 详情和安装指引 (Priority: P1)

用户点击某个 Skill 卡片后进入详情页，查看 Skill 的完整信息，包括名称、描述、版本、作者、安装方式、使用说明、截图/示例、下载量、更新日期等。页面提供清晰的安装指引（命令行安装和 zip 包安装两种方式）。

**Why this priority**: 详情页是用户决策是否安装的关键页面，与浏览功能同等重要，共同构成最小可用产品。

**Independent Test**: 通过直接访问某个 Skill 详情页 URL，验证所有信息正确展示，安装指引可复制。

**Acceptance Scenarios**:

1. **Given** 用户在列表页点击某个 Skill 卡片, **When** 进入详情页, **Then** 展示 Skill 完整信息（名称、描述、版本、作者、标签、下载量、更新时间）
2. **Given** 用户查看安装指引区域, **When** 切换安装方式 Tab, **Then** 分别展示命令行安装命令（可一键复制）和 zip 包下载按钮（可点击下载）
3. **Given** Skill 有使用说明/README, **When** 页面加载, **Then** 以 Markdown 渲染展示使用说明内容，支持表格、代码块、列表等常用格式
4. **Given** 用户点击 zip 包下载按钮, **When** 下载成功, **Then** 该 Skill 的下载量计数 +1

---

### User Story 3 - 用户注册登录和个人中心 (Priority: P2)

用户可以通过邮箱+密码注册账号并登录。登录后可以访问个人中心，查看自己提交的 Skill 列表、审核状态，以及编辑个人资料（昵称、头像、简介）。

**Why this priority**: 用户系统是 Skill 提交和管理功能的前置依赖，但浏览功能不需要登录，因此优先级略低于浏览和详情。

**Independent Test**: 完成注册、登录、查看个人中心、编辑个人资料的完整流程。

**Acceptance Scenarios**:

1. **Given** 未注册用户, **When** 填写邮箱和密码点击注册, **Then** 创建账号成功并自动登录，跳转到首页
2. **Given** 已注册用户, **When** 输入正确的邮箱和密码点击登录, **Then** 登录成功，页面头部显示用户信息
3. **Given** 已登录用户, **When** 访问个人中心, **Then** 展示个人资料、已提交的 Skill 列表及其审核状态
4. **Given** 已登录用户, **When** 编辑昵称/简介并保存, **Then** 个人资料更新成功
5. **Given** 用户输入错误的密码, **When** 点击登录, **Then** 提示"邮箱或密码错误"

---

### User Story 4 - Skill 提交与管理 (Priority: P2)

已登录用户可以提交新的 Skill，填写名称、描述、分类、版本、安装方式、使用说明等信息。提交后进入审核流程，管理员在后台审核通过后 Skill 才会在平台公开展示。用户可以在个人中心查看审核状态、修改或删除自己提交的 Skill；管理员可以修改和删除平台上所有 Skill。

**Why this priority**: Skill 提交是平台内容生态的来源，但初期可以由管理员手动录入数据，因此优先级为 P2。

**Independent Test**: 登录后提交一个 Skill，验证审核流程（待审核→通过/拒绝），验证通过后在列表页可见；验证作者可编辑/删除自己的 Skill；验证管理员可编辑/删除任意 Skill。

**Acceptance Scenarios**:

1. **Given** 已登录用户, **When** 点击"提交 Skill"并填写完整信息后提交, **Then** Skill 创建成功，状态为"待审核"
2. **Given** 管理员登录后台, **When** 查看待审核列表并点击"通过", **Then** Skill 状态变为"已发布"，在前台列表页可见
3. **Given** 管理员审核时, **When** 点击"拒绝"并填写拒绝原因（拒绝原因最长500字）, **Then** Skill 状态变为"已拒绝"，提交者在个人中心可看到拒绝原因；**And** 拒绝原因为必填项，不填写则无法提交
4. **Given** 管理员审核时, **When** 选择"拒绝"但未填写拒绝原因, **Then** 提示"请填写拒绝原因"，拒绝操作被阻止
5. **Given** 用户有已发布的 Skill, **When** 点击"更新"并修改信息提交, **Then** 系统创建新版本记录进入待审核，旧版本保持已发布状态在广场可见，新版本号必须大于当前版本号
6. **Given** 管理员在审核后台查看待审核 Skill, **When** 点击某个 Skill 的"查看"按钮, **Then** 弹出详情弹窗，显示该 Skill 的完整信息（图标、名称、描述、内容、版本、分类、作者、安装命令、zip 包信息等），界面样式清晰，参照提交 Skill 的表单样式，便于审核判断
7. **Given** 管理员审核通过某个新版本, **When** 点击"通过", **Then** 新版本变为已发布，旧版本变为已取代状态，广场显示新版本；审核记录中保留两版本的变更历史
8. **Given** 管理员审核拒绝某个新版本, **When** 点击"拒绝"并填写原因（拒绝原因最长500字，为必填项）, **Then** 新版本状态变为已拒绝，旧版本保持已发布状态不变
9. **Given** 用户在个人中心查看自己提交的 Skill, **When** 查看某个 Skill, **Then** 可以看到该 Skill 的所有版本记录及各自状态
10. **Given** 用户提交 Skill 时缺少必填字段, **When** 点击提交, **Then** 提示具体缺少的字段信息
11. **Given** 已登录用户在个人中心查看自己的 Skill, **When** 点击"编辑", **Then** 进入编辑页面可修改 Skill 信息，保存后重新进入审核流程
12. **Given** 已登录用户在个人中心查看自己的 Skill, **When** 点击"删除"并确认, **Then** Skill 被永久删除，从列表中移除
13. **Given** 管理员在管理后台的 Skill 管理区块, **When** 查看所有 Skill 列表并点击"编辑"或"删除", **Then** 管理员可以修改或删除任意 Skill，不受作者限制，不受审核状态限制
14. **Given** 普通用户尝试修改或删除他人的 Skill, **When** 发起请求, **Then** 系统返回 403 Forbidden，操作被拒绝
15. **Given** 用户更新已发布的 Skill 时, **When** 填写的新版本号不大于当前版本号, **Then** 提示"新版本号必须大于当前版本号"，提交被拒绝

---

### Edge Cases

- 搜索关键词无匹配结果时，展示"未找到相关 Skill"的友好提示和推荐内容
- 用户注册时邮箱已被使用，提示"该邮箱已注册"
- Skill 详情页访问不存在的 Skill ID 时，展示 404 页面
- 管理员拒绝 Skill 后，用户修改后可重新提交审核
- 并发提交相同名称的 Skill 时，系统应正确处理
- 首页各区块（热榜/推荐/最新）返回的是独立子集，数据可能有重叠；首页展示的 Skill 总数应去重后计数
- 用户上传超大文件（如 Skill 图标/截图）时，限制文件大小并提示
- 用户删除 Skill 时，需二次确认弹窗防止误操作
- 用户尝试编辑/删除他人 Skill 时，前端不展示操作按钮，后端返回 403
- Skill 未上传 zip 包时，详情页不展示 zip 包下载按钮
- zip 包上传格式校验：仅允许 .zip 格式，限制文件大小（最大 50MB）
- 用户更新已发布的 Skill 时，新版本号必须大于当前版本号，否则提示错误
- 用户更新已发布的 Skill 时，系统创建新版本记录，旧版本保持已发布状态在广场可见
- 管理员审核通过新版本后，旧版本变为"已取代"状态，不再在广场显示
- 用户提交 Skill 更新时，系统通过 parent_id 形成版本链，可追溯版本历史
- 所有时间显示格式统一为"年-月-日 时:分:秒"（如 2024-01-15 14:30:00），包括提交时间、更新时间、审核时间等

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供首页展示下载热榜、推荐、最近上新三个维度的 Skill 列表
- **FR-002**: 系统 MUST 支持按关键词搜索 Skill（搜索名称和描述）
- **FR-003**: 系统 MUST 支持按分类标签筛选 Skill
- **FR-004**: 系统 MUST 提供 Skill 详情页，展示完整信息和安装指引
- **FR-005**: 系统 MUST 支持用户通过邮箱+密码注册和登录
- **FR-006**: 系统 MUST 提供个人中心，展示用户资料和已提交的 Skill
- **FR-007**: 系统 MUST 支持已登录用户提交新 Skill
- **FR-008**: 系统 MUST 实现 Skill 审核流程（待审核→通过/拒绝）
- **FR-009**: 系统 MUST 提供管理员后台进行 Skill 审核管理
- **FR-010**: 系统 MUST 支持 Skill 列表分页加载
- **FR-011**: 系统 MUST 支持 Skill 版本更新（更新后需重新审核）
- **FR-018**: 系统 MUST 允许已登录用户修改自己提交的 Skill，修改后重新进入审核流程
- **FR-019**: 系统 MUST 允许已登录用户删除自己提交的 Skill（需二次确认）
- **FR-020**: 系统 MUST 允许管理员修改和删除平台上所有 Skill
- **FR-021**: 系统 MUST 对编辑/删除操作进行权限校验：普通用户只能操作自己的 Skill，管理员可操作任意 Skill，越权操作返回 403
- **FR-022**: 系统 MUST 支持已登录用户在个人中心修改自己的密码（旧密码验证+新密码设置）
- **FR-023**: 系统 MUST 支持管理员在后台修改任意用户的密码（无需旧密码验证）
- **FR-024**: Markdown 渲染 MUST 支持 GFM 表格扩展（详情页使用说明渲染）
- **FR-012**: 系统 MUST 对用户密码进行加密存储
- **FR-013**: 系统 MUST 使用 JWT 进行用户认证
- **FR-014**: 安装指引 MUST 支持一键复制功能（命令行安装命令）
- **FR-015**: 系统 MUST 支持 Skill zip 包上传（提交时上传，仅允许 .zip 格式，最大 50MB）
- **FR-016**: 系统 MUST 提供 Skill zip 包下载接口，下载文件名使用 Skill 名称（如 `{skill名称}.zip`），下载成功后下载量计数 +1
- **FR-017**: 前端 MUST 提供自定义 favicon 图标（`public/favicon.svg`）

### Key Entities

- **User**: 平台用户，包含邮箱、密码哈希、昵称、头像URL、简介、角色（普通用户/管理员）、注册时间
- **Skill**: 技能条目，包含名称、描述、分类、版本、安装方式（命令行/zip包）、使用说明（Markdown）、图标URL、zip包路径、作者（关联User）、下载量、审核状态（pending/approved/rejected/superseded）、父版本ID（parent_id，用于版本链）、创建/更新时间
- **Category**: 分类标签，包含名称、描述、排序权重
- **Review**: 审核记录，包含审核人、审核结果、拒绝原因、审核时间、关联Skill

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 首页加载时间不超过 3 秒
- **SC-002**: 搜索结果响应时间不超过 1 秒
- **SC-003**: 用户可以在 3 分钟内完成注册并提交第一个 Skill
- **SC-004**: Skill 详情页信息完整率 100%（所有必填字段均展示）
- **SC-005**: 审核流程状态流转正确率 100%
- **SC-006**: 系统支持 500 并发用户正常访问

## Assumptions

- 用户使用现代浏览器（Chrome、Firefox、Edge、Safari 最新两个版本）
- 初期不需要移动端 App，Web 端做好响应式适配即可
- 不需要第三方 OAuth 登录（如微信、GitHub），仅邮箱+密码
- Skill 的实际安装过程在用户本地完成，平台提供安装指引文本和 zip 包下载
- 不需要 Skill 的在线运行/预览功能
- 管理员账号通过数据库直接创建，不需要管理员注册功能
- 文件存储（图标、截图）使用本地文件系统，不使用云存储
- 数据库使用 MySQL，不使用 Docker 部署
